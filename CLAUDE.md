# CLAUDE.md — Self-Healing MLOps Pipeline

## Project Overview
A self-healing retraining pipeline for a fraud detection model. When data drift
is detected in production traffic, the system automatically retrains and saves a
new model version — no human intervention required.

## Tech Stack
- **Feature Store**: Feast (local provider, SQLite online store, file-based offline store)
- **ML Framework**: scikit-learn (RandomForestClassifier)
- **Drift Detection**: Evidently AI (DataDriftPreset)
- **CI/CD**: GitHub Actions (cron-scheduled every 6 hours)
- **Python**: 3.11+

## Architecture Rules

### Directory Layout
```
feature_repo/          — Feast definitions (feature_store.yaml, features.py)
src/                   — All application code
  generate_data.py     — Synthetic data generator (reference + current)
  monitor.py           — Evidently drift detection (reference vs current)
  retrain_trigger.py   — Self-healing orchestrator (monitor → decide → retrain)
data/                  — Runtime artifacts (CSVs, drift reports) — gitignored
models/                — Timestamped .pkl model files — gitignored
.github/workflows/     — CI/CD pipeline definitions
```

### Key Conventions
1. **Drift threshold is 0.3** — defined in `src/retrain_trigger.py::DRIFT_THRESHOLD`.
   Drift score = fraction of feature columns that drifted (0.0–1.0).
2. **Models are versioned by UTC timestamp** — format: `fraud_model_YYYYMMDDTHHMMSSz.pkl`
3. **All modules run as `python -m src.<module>`** — never run files directly with path.
4. **Feature columns are shared** between monitor.py and retrain_trigger.py:
   `user_transaction_count`, `user_transaction_amount_avg`, `user_transaction_amount_max`
5. **Reference data = historical stable distribution**; current data = new production logs.
6. **Data artifacts are gitignored** — they are regenerated by the pipeline each run.

### Self-Healing Flow
```
generate_data → monitor.py (Evidently drift report)
                    ↓
              drift_score > 0.3?
              ├── YES → "Drift Detected!" → train_model() → save timestamped .pkl
              └── NO  → "Model is healthy." → exit clean
```

### Feast Feature Store
- Entity: `user_id` (join key)
- FeatureView: `user_transaction_features` with TTL of 1 day
- Source: `data/user_transactions.parquet` (parquet file source)
- Config: `feature_repo/feature_store.yaml`

### GitHub Actions Workflow
- Trigger: cron `0 */6 * * *` + manual workflow_dispatch
- Artifacts uploaded: `data/drift_report.json`, `models/*.pkl`
- Runner: ubuntu-latest, Python 3.11, pip cache enabled

## Common Commands
```bash
pip install -r requirements.txt          # Install dependencies
python -m src.generate_data              # Generate reference + drifted current data
python -m src.monitor                    # Run drift detection only
python -m src.retrain_trigger            # Full self-healing loop
```

## Do NOT
- Put model files or CSV data into git (they are gitignored for a reason)
- Change the drift threshold without updating this doc
- Import modules with file paths — always use `from src.<module>` or `python -m`
- Add new feature columns without updating BOTH monitor.py and retrain_trigger.py
